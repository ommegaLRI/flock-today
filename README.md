# Flock

Flock is the owner-only editing capsule for websites generated by the private Stitch migration system.

It deliberately does one thing:

> Select a Stitch section, describe a change, regenerate that section, and see the result through Astro HMR.

Flock does not migrate sites, reinterpret the Stitch contract, generate an alternative project, run review workflows, or appear in production builds.

## Current phase

This repository implements the complete capsule boundary:

- verifies a Stitch-generated Astro project;
- installs one Astro integration;
- injects the owner capsule only during `astro dev`;
- discovers section files from `data-section-id` and `data-stitch-role="section"`;
- reads focused context from `.stitch` without modifying it;
- provides a single `generateSection(context, instruction)` seam;
- safely replaces exactly one section file;
- supports one in-session revert;
- emits no Flock runtime during `astro build`.

The built-in AI itself is intentionally not implemented yet. Without a generator, section selection and inspection work and the capsule reports that generation is unavailable.

## Install in a Stitch project

```bash
npm install --save-dev @flock/capsule
npx flock install
npm run dev
```

The installer makes one idempotent change to `astro.config.*`:

```js
import flock from '@flock/capsule';

export default defineConfig({
  integrations: [flock()],
  // existing Stitch configuration
});
```

Open the site and choose **Edit with Flock**.

## AI seam

The future built-in AI plugs into one function:

```ts
import flock from '@flock/capsule';

export default defineConfig({
  integrations: [
    flock({
      async generateSection({ instruction, context }) {
        // Return the complete Astro source for context.section.id.
        // The result must preserve:
        // data-section-id="..."
        // data-stitch-role="section"
        return generatedAstroSource;
      },
    }),
  ],
});
```

The context contains only section-local material:

- current Astro source;
- the matching section contract;
- section facts and occurrences;
- applicable recipes and design tokens;
- section review/failure context when present;
- the original Stitch crop path;
- the owner instruction and current route.

## Ownership boundary

Flock treats these as immutable Stitch artifacts:

```text
.stitch/manifest.json
.stitch/contract.json
.stitch/provenance.json
.stitch/run.json
.stitch/visuals/**
.stitch/failures/**
```

Flock writes only the selected generated `.astro` section file. It does not claim that a modified section remains a valid projection of the original Stitch contract.

## Production

A normal production build injects no Flock script, endpoint, panel, credential, or Stitch artifact:

```bash
npm run build
```

Flock is an owner-development tool only.
